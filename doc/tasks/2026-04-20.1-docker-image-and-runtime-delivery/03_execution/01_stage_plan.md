# 阶段计划

## Stage 1：冻结 Spec

- 明确镜像构建路径、容器内数据目录、运行时入口、端口映射与安全边界。
- 明确哪些部署层内容仍留给后续任务。

## Stage 2：落地镜像文件

- 新增 `Dockerfile`。
- 新增 `.dockerignore`。
- 视需要补充最少量仓库入口更新，例如 README 或 `justfile` 中的 Docker 入口。

## Stage 3：验证本地与 VPS 运行路径

- 验证镜像可构建。
- 验证缺失 `APP_TOKEN` 时启动失败。
- 验证带数据目录挂载时 `/healthz`、登录状态与页面入口可用。
- 验证本地重启容器后认证文件与 spec 缓存可复用。
