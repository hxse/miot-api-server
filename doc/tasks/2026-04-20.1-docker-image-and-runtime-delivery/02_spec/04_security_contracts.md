# 安全 Contract

## 镜像安全

- 镜像中不得内置 `APP_TOKEN`。
- 镜像中不得内置真实认证文件。
- 镜像中不得依赖 query string 传 token。

## 运行时安全

- token 只能通过运行时环境变量注入。
- 容器内部监听 `0.0.0.0` 只服务于端口映射，不等于默认公网开放。
- 宿主机端口映射必须由操作者显式声明。
- 当前任务不默认开放 80/443，也不内置反向代理。

## 数据安全

- 认证文件与 spec 缓存必须位于容器内稳定数据目录下。
- 默认建议只挂载 `/data/mijia` 对应的 volume，不把整个项目目录映射进容器。
- 删除容器不应被误认为会删除 volume；需要彻底清空登录状态时，必须额外处理对应数据卷。
