# 任务描述

本任务作为当前仓库第一阶段的唯一 task snapshot，统一承接三类内容：立项与技术路线选择、`uv run` 启动与最小公网安全基线、以及当前设备能力导向控制模型。当前源码与本 task 共同构成第一阶段正式真值。

# 任务级别

本任务定级为 `A` 类。原因是它同时覆盖项目真值下沉、启动与安全 contract、公开 HTTP contract、登录页与设备控制页设计、运行时状态模型与同步 provider 执行模型约束，影响公开接口、页面交互、运行时行为与后续任务基线。

# 任务范围

## In Scope

- 依据 `todo.md` 分析第一阶段技术可行性与主要风险。
- 对比两条技术路线：复用 `Do1e/mijia-api` 与手动解析 `XiaoMi/ha_xiaomi_home`。
- 把业务运行时真值从 `AGENTS.md` 下沉到 task。
- 冻结 `uv run` 启动方式、`APP_TOKEN` 失败语义、认证文件与 spec 缓存路径约束。
- 冻结公网 VPS 场景下的最小应用内安全基线。
- 落地第一阶段最小 FastAPI 闭环，包括专用登录与设备控制页、登录初始化、设备发现、设备能力描述、设备级 `power` 控制、测试重置与健康检查。
- 冻结设备为一级资源、能力为子路由的控制模型。
- 冻结认证文件与设备 spec 缓存的本地持久化约束。
- 明确当前同步 provider 业务路由统一使用 `def` 承接。
- 统一 task 索引，只保留当前这一份第一阶段 task。

## Out of Scope

- 不打通 Docker、本地容器调试或 VPS `docker run` 部署链路。
- 不冻结 HTTPS、反向代理、限流与公网防火墙收口等部署层安全细节。
- 不扩展除 `power` 之外的正式控制能力，也不设计通用 MIoT 属性读写 DSL。
