# 安全 Contract

## 启动安全

- 服务在 `APP_TOKEN` 缺失、为空字符串或仅包含空白字符时不得启动。
- 服务不得为了“方便启动”而自动生成默认 token。
- `APP_TOKEN` 必须由操作者显式提供；当前任务允许通过环境变量注入。
- 只有 `just uv-run` 这个当前正式调试入口默认绑定 `0.0.0.0`。
- 除 `just uv-run` 外，其他未显式传 `--host` 的启动入口默认保持 `127.0.0.1`。
- `just uv-run` 的 `0.0.0.0` 绑定只服务于当前阶段测试便利，不得被解释成应用通用默认 host。

## HTTP 暴露面

- `/login` 作为登录与设备控制壳页存在；页面本身允许匿名访问，但页面内实际调用的登录接口、设备接口与控制接口仍必须携带 Bearer token。
- `/docs` 与 `/redoc` 只作为文档壳页存在；未提供 token 时不得加载 OpenAPI schema 内容。
- `/openapi.json` 不作为默认公开入口，必须在 Bearer 鉴权通过后才能访问。
- `GET /healthz` 只返回最小存活信息，不返回配置、账号、设备或认证上下文。
- 除明确豁免的健康检查路径、登录壳页与文档壳页外，其余请求默认都需要 Bearer token。

## 认证口径

- 正式认证头口径冻结为 `Authorization: Bearer <TOKEN>`。
- token 不允许通过 query string 传递。
- 认证失败默认返回明确的未授权错误，而不是静默放行或兼容回退。
- 文档页加载 schema 时，也必须沿用同一 Bearer token 口径。

## 当前任务边界

- 本任务只冻结应用内最小安全基线，不冻结 HTTPS、反向代理、限流、云防火墙和公网端口收口细节。
- 这些部署层安全能力必须在后续 Docker 或 VPS 部署任务中继续冻结。
