# 最终验收

## 验收结论

- 实现与 `02_spec` 一致。
- curl/API 业务入口不再因 `api.available=false` 提前阻断可刷新的旧认证文件。
- 业务认证刷新路径不再直接调用底层 `_refresh_token()`，小米账号服务请求和回调请求都带显式 timeout。
- `APP_TOKEN` Bearer 鉴权、扫码登录流程、认证文件路径、Docker volume、设备发现与 power 控制语义未改动。
- 离线测试覆盖刷新成功、刷新失败、刷新网络异常、无认证文件、损坏认证文件、缺字段认证文件、可用认证不刷新、刷新 timeout 参数，以及 `/api/devices` 路由函数回归。

## 残余风险

- 本任务未访问真实小米云，无法证明真实账号在小米风控、网络波动或云端协议变化下必然刷新成功。
- 当前实现继续依赖 `mijiaAPI==3.0.5` 的私有认证字段、`_handle_ret()`、`_save_auth_data()` 与 `_init_session()` 行为；后续升级该依赖需要单独验证登录链路。
- `/api/auth/login/start` 的扫码流程保持原样；本任务没有扩展损坏认证文件下的网页登录恢复体验。
